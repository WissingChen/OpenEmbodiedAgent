# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.0.5] - 2026-03-27

### Added
- **多本体协同 (Fleet Mode)**: 引入了 `Fleet Watchdog`，支持在同一个工作区拓扑下同时管理和调度多个异构机器人（如机械臂与四足机器狗协同）。
- **外部插件动态加载机制**: 新增 `hal/plugins.py`，允许通过命令行参数动态加载外部硬件驱动（如真实的 ReKep 机械臂），无需修改核心代码即可扩展新硬件支持。
- **语义导航工具 (`SemanticNavigationTool`)**: 允许 Agent 将高层语义目标（如“移动到厨房的桌子旁”）解析为具体的物理坐标，并生成底层的导航动作指令。
- **多模态感知管线 (`PerceptionService`)**: 
  - 新增 `GeometryPipeline` 处理点云和里程计数据。
  - 新增 `SegmentationPipeline` 处理图像语义分割。
  - 新增 `FusionPipeline` 将几何与语义信息融合为 Scene-Graph，并由 `EnvironmentWriter` 写入 `ENVIRONMENT.md`。
- **用户文档模板**: 在 `docs/` 目录下新增了《用户开发手册》(`USER_DEVELOPMENT_GUIDE.md`) 和《使用手册》(`USER_MANUAL.md`) 的草案模板。
- **Changelog 规范**: 确立了每次更新在 `docs/changelog/` 目录下记录版本变更的规范。

### Changed
- **README_zh.md 更新**: 全面更新了中文版 README，反映了 v2.1.0 的最新特性（多本体支持、插件机制、语义导航等），并更新了 Roadmap 的完成状态。
- **架构升级**: 进一步深化了“认知-物理解耦”的架构设计，完善了硬件抽象层 (HAL) 的接口定义，使其更好地支持动态加载和多设备协同。

### Fixed
- 修复了早期版本中部分文档描述不够严谨的问题，统一了学术化和工程化的表达风格。
